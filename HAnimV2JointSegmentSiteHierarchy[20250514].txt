expalation of hierarchy notation: 
joint_a  x y z      #Jnn
  segment_a      #Bnn
    [segment_a  joint_a-to-joint_b]
  joint_b  x y z      #Jnn 
    segment_b      #Bnn
      [segment_b  joint_b-to-segment_b_tip]
        (segment_b_tip  x y z)      #Snn 
          {joint_b-to-segment_b_tip} 
...
  
    <!-- 
      0 0 0 at floor between feet, 
      default I pose (relaxed attention) 
      model gaze toward +Z, 
      +X to model left, +Y up, 
      right-hand rule. 
      All Joint and Site locations relative to 0 0 0 -->


joint_n  x y z represents a Joint node in the hierarchy 
  and gives its coordinates relative to 0 0 0. 
segment_n represents a connection between Joint nodes. 
[segment_n  from ...-to-...] describes a connection 
  between Joint nodes or Joint node to end effector Site.
(segment_n_tip  x y z) represents a Site node in the hierarchy  
  and gives its coordinates relative to 0 0 0. 
{joint_b-to-segment_b_tip} describes the connection 
  between a Joint and a Site. 

So, in the example below, the Segment 'sacrum' is a 
child of root and connects with two joints, sacro and vl5. 
From the sacro joint, the pelvis connects-to-
right and left hips. The pelvis includes several Site nodes. 
  
The Joint and Site x y z coordinates listed here are from 
hanim v2 Part 1 Annex A Tables listing of example 
Joint centers and Site locations.
